Git

发表于2026-06-26|更新于2026-06-26|前端学习笔记

定义

概念：一个免费开源,分布式的代码版本控制系统,帮助开发团队维护代码

作用：记录代码内容,切换代码版本,多人开发时高效合并代码内容

Git 配置 用户信息

配置用户名和邮箱，应用在每次提交代码版本时表明自己身份

命令

Git 仓库

定义：记录文件状态内容的地方，存储着修改的历史记录

创建

  • 把本地文件夹转换为 Git 仓库
  • 从其他服务器上克隆 Git 仓库

Git 的三个区域

工作区：实际开发时操作的文件夹

暂存区：保存之前的准备区域（暂存改动过的文件）

版本库：提交并保存暂存区中的内容，产生一个版本快照

命令 作用
git add 文件名（相对路径，相对于工作区的根目录） 暂存指定文件
git add . 暂存所有改动的文件
git commit -m “注释说明” 提交并保存，产生版本快照
git ls-files 查看当前项目下暂存区的文件

Git 文件状态

Git 文件 2 种状态：

  • 未跟踪：新文件，从未被Git管理过（即未被添加到暂存区）
  • 已跟踪：Git 已经知道和管理的文件（已被添加到暂存区）
文件状态 概念 场景
未跟踪（U) 从未被 Git 管理过 新文件
新添加（A） 第一次被 Git 暂存 之前版本记录无此文件
未修改（’’） 三个区域统一 修改保存后
已修改（M） 工作区内容发生改变，即和暂存区内容不一致 此文件添加到暂存区后修改了内容，那么该文件在工作区的状态就为已修改，同理可知暂存区的已修改状态场景

git status -s 命令可以查看文件状态

例如：MM login/index.html

其中第一列是该文件在暂存区的状态，第二列是该文件在工作区的状态

Git 暂存区使用

  1. 暂存区：暂时存储,可以临时恢复代码内容,与版本库解耦
  2. 可用暂存区覆盖工作区的某一文件
  1. 从暂存区移除文件

Git 回退版本

查看提交到 Git 版本库的历史记录

查看提交到 Git 版本库的完整历史记录

回退命令

  • git reset –soft 版本号：会将该版本号的所有文件都覆盖回工作区和暂存区，工作区和暂存区的其他文件内容无变化，状态改为新添加（A）
  • git reset –hard 版本号：会将该版本号的所有文件都覆盖回工作区和暂存区，暂存区的其他文件被删除，然后会把暂存区和工作区的所有文件进行对比，从工作区中删除暂存区未存在的文件
  • git reset –mixed 版本号：会将该版本号的所有文件都覆盖回工作区和暂存区，暂存区的其他文件被删除，工作区的其他文件保留，状态改为未跟踪（U）

注意1：只有记录在版本库的提交记录才能恢复

注意2：回退后，继续修改->暂存->提交操作即可产生新的提交记录

删除文件

删除工作区的文件，然后再添加到暂存区，后提交到版本库即可实现删除文件

忽略文件

概念：.gitignore 文件可以让 git 彻底忽略跟踪指定文件

目的：让 git 仓库更小更快，避免重复无意义的文件管理

步骤：

  1. 项目根目录新建 .gitignore 文件
  2. 填入相应配置来忽略指定文件

注意：如果文件已经被暂存区跟踪过，可以从暂存区移除即可

分支

分支-创建和切换

  1. 概念：本质上是指向提交节点的可变指针，默认名字是master
  2. 注意：HEAD 指针影响工作区/暂存区的代码状态
  3. 场景：开发新需求/修复Bug，保证主线代码随时可用，多人协同开发提高效率
  4. 命令
  • 创建分支命令
  • 切换分支命令
  • 查看所有分支名
  • 创建并立刻切换分支

分支-合并与删除

  1. 切回要合入的分支上
  1. 合并其他分支
  1. 删除合并后的分支指针

分支-合并与提交

合并提交：发生于原分支产生了新的提交记录后，再合并回去时发生，自动使用多个快照记录合并后产生一次新的提交。合并的步骤不变

分支-合并冲突

概念：不同分支中，对同一个文件的同一部分修改，Git 无法干净的合并，产生合并冲突

解决：手动选择保留哪个版本的冲突代码

Git 远程仓库

Git 远程仓库-推送

概念：托管在因特网或其他网络中的你的项目的版本库

作用：保存版本库的历史记录，多人协作

创建：公司自己服务器/第三方托管平台（Gitee，GitLab，GitHub…）

命令：

  • 本地 Git 仓库添加远程仓库原点地址
  • 本地 Git 仓库推送版本记录到远程仓库

如果本地和远程分支名一致，可以简写（只写一个分支名即可）

  • 删除本地与远程的连接
  • 查看本地与远程的连接
  • 当远程仓库和本地仓库第一次进行推送，并且远程仓库不为空，此时就需要强制拉取

多人协同开发

步骤：

  1. 程序员1开发代码 -> 工作区 -> 添加到暂存区 -> 提交到版本库 -> 拉取远程仓库的最新版本（可选，本地版本就是最新记录可以不用拉取）-> 推送到远程仓库
  2. 程序员2拉取远程仓库的最新版本 -> 开发代码 -> 工作区 -> 添加到暂存区 -> 提交到版本库 -> 拉取远程仓库的最新版本（可选，本地版本就是最新记录可以不用拉取）-> 推送到远程仓库
